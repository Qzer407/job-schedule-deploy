# 告警系统模块设计

## 1. 模块概述

告警系统模块负责任务执行异常时的通知功能，支持多渠道告警（邮件、企业微信、钉钉），采用策略模式实现不同渠道的告警发送。

## 2. 功能设计

### 2.1 功能列表

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 告警配置管理 | 配置各渠道的告警参数 | P0 |
| 邮件告警 | 通过邮件发送告警通知 | P0 |
| 企业微信告警 | 通过企业微信机器人发送通知 | P0 |
| 钉钉告警 | 通过钉钉机器人发送通知 | P0 |
| 告警记录查询 | 查询历史告警记录 | P1 |
| 告警开关 | 控制各渠道告警的启用/禁用 | P1 |

### 2.2 核心业务流程

```
任务执行失败
      │
      ▼
┌─────────────┐
│ AlarmService│
│ 检测告警条件 │
└──────┬──────┘
       │
       ▼
┌──────────────────────────────────────────┐
│      遍历启用的告警渠道                   │
└──────────────────────────────────────────┘
       │
       ├──► EmailAlarmHandler ──► 发送邮件
       │
       ├──► WeChatAlarmHandler ──► 企业微信通知
       │
       └──► DingTalkAlarmHandler ──► 钉钉通知
```

## 4. API接口设计

### 4.1 接口列表

| HTTP方法 | API路径 | 功能描述 | Controller文件 |
|----------|---------|----------|----------------|
| GET | `/api/v1/alarms/config` | 查询所有告警配置 | AlarmController |
| GET | `/api/v1/alarms/config/{channelType}` | 查询单个告警配置 | AlarmController |
| POST | `/api/v1/alarms/config` | 创建告警配置 | AlarmController |
| PUT | `/api/v1/alarms/config/{channelType}` | 更新告警配置 | AlarmController |
| DELETE | `/api/v1/alarms/config/{channelType}` | 删除告警配置 | AlarmController |
| GET | `/api/v1/alarms/records` | 查询告警记录 | AlarmController |
| POST | `/api/v1/alarms/test/{channelType}` | 测试告警渠道 | AlarmController |

### 4.2 请求响应示例

**创建告警配置请求:**
```json
{
  "channelType": "EMAIL",
  "channelName": "邮件告警",
  "channelConfig": "{\"smtpHost\":\"smtp.example.com\",\"smtpPort\":465,\"username\":\"admin@example.com\",\"password\":\"xxx\",\"fromEmail\":\"alarm@example.com\",\"toEmails\":\"user1@example.com,user2@example.com\"}",
  "enabled": true
}
```

**响应:**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "channelType": "EMAIL",
    "channelName": "邮件告警",
    "enabled": true,
    "createTime": "2026-05-13T10:00:00"
  }
}
```

## 5. 数据库设计

### 5.1 核心数据表

#### t_alarm_config（告警配置表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | 配置ID |
| channel_type | VARCHAR(50) | NOT NULL UNIQUE | 渠道类型(EMAIL/WX_DING/DING_TALK) |
| channel_name | VARCHAR(200) | | 渠道名称 |
| channel_config | TEXT | | 渠道配置(JSON格式) |
| enabled | TINYINT(1) | DEFAULT 1 | 是否启用 |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

#### t_alarm_record（告警记录表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | 记录ID |
| task_id | BIGINT | | 关联任务ID |
| channel_type | VARCHAR(50) | NOT NULL | 告警渠道类型 |
| alarm_level | VARCHAR(20) | | 告警级别(INFO/WARN/ERROR) |
| alarm_title | VARCHAR(200) | NOT NULL | 告警标题 |
| alarm_content | TEXT | | 告警内容 |
| send_status | TINYINT | DEFAULT 0 | 发送状态(0:待发送,1:成功,2:失败) |
| error_msg | TEXT | | 错误信息 |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
