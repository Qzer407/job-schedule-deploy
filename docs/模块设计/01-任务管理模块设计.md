# 任务管理模块设计

## 1. 模块概述

任务管理模块是系统的核心功能模块，负责定时任务的创建、编辑、删除、启动、停止、触发等操作，基于XXL-JOB实现任务调度。

## 2. 功能设计

### 2.1 功能列表

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 任务列表查询 | 分页查询任务列表，支持条件筛选 | P0 |
| 任务创建 | 创建新的定时任务 | P0 |
| 任务编辑 | 修改已有任务配置 | P0 |
| 任务删除 | 删除任务（需确认） | P0 |
| 任务启动 | 启动任务调度 | P0 |
| 任务停止 | 停止任务调度 | P0 |
| 任务触发 | 手动触发一次任务执行 | P1 |
| 任务日志 | 查询任务执行日志 | P1 |

### 2.2 核心业务流程

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  前端请求   │ ──► │ TaskController│ ──► │ TaskService │
└─────────────┘     └──────────────┘     └──────┬──────┘
                                                │
                        ┌───────────────────────┼───────────────────────┐
                        ▼                       ▼                       ▼
              ┌───────────────┐      ┌─────────────────┐      ┌─────────────────┐
              │ XxlJobAdminService│    │   TaskInfoMapper   │    │   TaskLogMapper   │
              │ (调用XXL-JOB API) │    │   (数据库操作)     │    │   (日志记录)      │
              └───────────────┘      └─────────────────┘      └─────────────────┘
```

## 4. API接口设计

### 4.1 接口列表

| HTTP方法 | API路径 | 功能描述 | Controller文件 |
|----------|---------|----------|----------------|
| GET | `/api/v1/tasks` | 分页查询任务列表 | TaskController |
| GET | `/api/v1/tasks/{id}` | 查询单个任务详情 | TaskController |
| POST | `/api/v1/tasks` | 创建任务 | TaskController |
| PUT | `/api/v1/tasks/{id}` | 更新任务 | TaskController |
| DELETE | `/api/v1/tasks/{id}` | 删除任务 | TaskController |
| POST | `/api/v1/tasks/{id}/start` | 启动任务 | TaskController |
| POST | `/api/v1/tasks/{id}/stop` | 停止任务 | TaskController |
| POST | `/api/v1/tasks/{id}/trigger` | 触发任务 | TaskController |

### 4.2 请求响应示例

**创建任务请求:**
```json
{
  "taskName": "数据同步任务",
  "jobGroup": "default",
  "jobDesc": "每日同步数据",
  "cronExpression": "0 0 2 * * ?",
  "executorHandler": "dataSyncHandler",
  "executorParam": "{\"source\":\"db\",\"target\":\"es\"}",
  "executorRouteStrategy": "FIRST",
  "executorBlockStrategy": "SERIAL_EXECUTION",
  "timeout": 300,
  "retryCount": 3,
  "status": 0
}
```

**响应:**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "taskName": "数据同步任务",
    "jobGroup": "default",
    "cronExpression": "0 0 2 * * ?",
    "createTime": "2026-05-13T10:00:00"
  }
}
```

## 5. 数据库设计

### 5.1 核心数据表

#### t_task_info（任务信息表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | 任务ID |
| task_name | VARCHAR(200) | NOT NULL | 任务名称 |
| job_group | VARCHAR(100) | NOT NULL | 执行器分组 |
| job_desc | VARCHAR(500) | | 任务描述 |
| cron_expression | VARCHAR(100) | NOT NULL | CRON表达式 |
| executor_handler | VARCHAR(200) | NOT NULL | 执行器Handler名称 |
| executor_param | TEXT | | 执行参数 |
| executor_route_strategy | VARCHAR(50) | | 路由策略 |
| executor_block_strategy | VARCHAR(50) | | 阻塞策略 |
| timeout | INT | DEFAULT 0 | 超时时间(秒) |
| retry_count | INT | DEFAULT 0 | 重试次数 |
| status | TINYINT | DEFAULT 0 | 状态(0:停止,1:运行) |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

#### t_task_log（任务日志表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | 日志ID |
| task_id | BIGINT | NOT NULL | 任务ID |
| trigger_time | DATETIME | NOT NULL | 触发时间 |
| start_time | DATETIME | | 开始时间 |
| end_time | DATETIME | | 结束时间 |
| execution_status | TINYINT | DEFAULT 0 | 执行状态 |
| execution_log | TEXT | | 执行日志 |
| error_msg | TEXT | | 错误信息 |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
