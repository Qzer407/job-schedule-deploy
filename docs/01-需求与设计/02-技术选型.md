## 2. 后端技术选型

### 2.1 语言与框架

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| Java | 21 | LTS 版本，性能稳定，生态成熟，适合企业级后端服务 |
| Spring Boot | 3.2.x | 社区成熟，生态完善，便于快速构建微服务 |

### 2.2 数据存储

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| MySQL | 8.0+ | 成熟稳定，支持事务，社区活跃，适合存储任务配置和执行日志 |
| Redis | 7.0+ | 缓存热点数据，加速调度响应，支持分布式锁 |

### 2.3 分布式组件

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| Redisson | 3.23.x | 基于 Redis 的分布式锁实现，支持多种锁类型 |
| XXL-JOB | 2.4.x | 成熟的分布式任务调度框架，支持分片、重试、告警等 |
| RabbitMQ | 3.12+ | 实现任务异步化，削峰填谷，提升并发处理能力 |

### 2.4 ORM框架

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| MyBatis-Plus | 3.5.x | 增强 MyBatis，提供便捷的 CRUD 操作，支持分页、乐观锁等 |

## 3. 前端技术选型

### 3.1 框架与构建

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| Vue | 3.4.x | 响应式框架，组合式 API，性能优秀 |
| Vite | 6.5.x | 快速构建，热更新，适合 Vue 项目 |

### 3.2 UI组件

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| Element Plus | 2.6.x | Vue 3 官方组件库，样式美观，组件丰富 |
| ECharts | 5.5.x | 强大的数据可视化能力，支持多种图表类型 |

## 4. 基础设施

### 4.1 容器化

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| Docker | 25.x | 环境一致性，便于部署和运维 |
| Docker Compose | 2.24.x | 简化多容器应用的部署 |

## 5. 技术选型决策矩阵

| 需求 | 技术选型 | 替代方案 | 选型依据 |
|-----|---------|---------|---------|
| 任务调度 | XXL-JOB | Quartz | XXL-JOB 提供更完善的分布式调度能力和监控功能 |
| 消息队列 | RabbitMQ | Kafka | RabbitMQ 更适合任务调度场景，消息可靠性更高 |
| 缓存 | Redis | Memcached | Redis 支持更多数据结构，可用于分布式锁 |
| 分布式锁 | Redisson | Zookeeper | Redisson 基于 Redis，性能更好，部署更简单 |
| 前端框架 | Vue 3 | React | 团队技术栈偏好，Vue 3 组合式 API 更灵活 |
