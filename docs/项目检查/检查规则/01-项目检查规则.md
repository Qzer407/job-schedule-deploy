# 项目检查规则

## 1. 检查时机

| 检查类型 | 触发条件 | 优先级 | 执行人 |
|---------|---------|-------|-------|
| **提交前检查** | 每次代码提交前 | 🔴 高 | 开发者 |
| **迭代结束检查** | 每个迭代完成后 | 🔴 高 | 负责人 |
| **发布前检查** | 正式发布前 | 🔴 高 | 负责人 + QA |
| **周度检查** | 每周一 | 🟡 中 | 负责人 |
| **里程碑检查** | 需求/设计/开发/测试 阶段结束 | 🟡 中 | 负责人 |

---

## 2. 检查报告输出规范

### 2.1 报告格式

检查报告需包含以下部分：

```markdown
# [项目名] - [检查类型] - [日期] 检查报告

## 1. 检查概况
- 检查时间：YYYY-MM-DD
- 检查范围：[简述]
- 检查人：[姓名]
- 上一次检查时间：[日期]

## 2. 检查结果汇总
- 检查总项数：X
- 通过项数：X
- 问题项数：X
- 严重问题数：X
- 中等问题数：X
- 轻微问题数：X

## 3. 问题清单
| 序号 | 问题描述 | 严重程度 | 位置 | 修复建议 | 修复状态 | 修复人 |
|-----|---------|---------|-----|---------|---------|-------|
| 1 | ... | 🔴 严重 | xxx.java | ... | 未修复 | - |

## 4. 改进建议

## 5. 上一次问题修复情况
```

### 2.2 问题严重程度定义

| 级别 | 标识 | 说明 | 处理时限 |
|-----|-----|-----|---------|
| **严重** | 🔴 | 阻塞功能、影响稳定性、安全问题 | 24小时内 |
| **中等** | 🟡 | 功能不完善、体验不佳 | 1周内 |
| **轻微** | 🟢 | 代码规范、文档完善 | 2周内 |

### 2.3 问题状态定义

| 状态 | 说明 |
|-----|-----|
| `未修复` | 问题已发现，未开始处理 |
| `修复中` | 正在修复该问题 |
| `待验证` | 修复完成，等待验证 |
| `已关闭` | 验证通过，问题已解决 |
| `已忽略` | 不影响功能，暂不处理 |

## 3. 检查范围

### 3.1 后端检查范围（Java/Spring Boot）

#### 3.1.1 代码完整性
- [ ] 主应用入口文件完整
- [ ] Controller 层：所有API接口完整
- [ ] Service 层：所有业务逻辑实现完整
- [ ] Mapper 层：所有数据访问完整
- [ ] Entity 层：所有实体类完整
- [ ] Config 层：配置类完整
- [ ] 依赖管理：pom.xml 配置正确
- [ ] 启动配置：application.yml 配置正确
- [ ] 工具类：常用工具类齐全

#### 3.1.2 API 设计
- [ ] 统一响应格式 (ApiResponse)
- [ ] 统一异常处理 (GlobalExceptionHandler)
- [ ] API 路径规范：/api/v1/module/...
- [ ] HTTP 方法正确使用
- [ ] 参数校验 (@Valid, @NotBlank 等)
- [ ] 分页参数规范 (pageNum, pageSize)

#### 3.1.3 数据库
- [ ] 所有实体对应有表结构
- [ ] schema.sql 完整
- [ ] 表名规范：t_xxx
- [ ] 字段名规范：snake_case
- [ ] 索引合理
- [ ] 外键约束正确

#### 3.1.4 安全
- [ ] 认证授权实现完整
- [ ] 密码加密存储 (BCrypt)
- [ ] JWT Token 验证
- [ ] 接口权限控制
- [ ] SQL 注入防护
- [ ] 敏感数据不打印日志

#### 3.1.5 测试
- [ ] 单元测试覆盖核心业务
- [ ] 测试用例可正常运行
- [ ] 集成测试规划

#### 3.1.6 文档
- [ ] API 文档集成 (Swagger/OpenAPI)
- [ ] 代码注释清晰

## 4. 详细检查项清单

### 4.1 后端详细检查项

| 检查项 | 标准 | 严重程度 |
|-------|------|---------|
| **项目结构** | 目录结构清晰分层明确 | 🟢 轻微 |
| **主应用类** | @SpringBootApplication 注解正确，main 方法存在 | 🔴 严重 |
| **Controller 层** | @RestController / @Controller，路径规范，参数校验 | 🟡 中等 |
| **Service 层** | @Service 注解，@Transactional 事务处理，业务逻辑完善 | 🟡 中等 |
| **Entity 层** | @TableName，@TableId，@TableField 注解正确 | 🟢 轻微 |
| **pom.xml** | 依赖完整，版本兼容 | 🔴 严重 |
| **application.yml** | 配置项完整，端口、数据库、XXL-JOB 等配置正确 | 🔴 严重 |
| **SecurityConfig** | 安全配置合理，白名单正确，JWT 拦截器配置 | 🔴 严重 |
| **JwtAuthenticationFilter** | Token 验证逻辑正确 | 🔴 严重 |
| **GlobalExceptionHandler** | 统一异常处理，返回 ApiResponse | 🟡 中等 |
| **OpenApiConfig** | Swagger 配置正确，访问路径 /swagger-ui/index.html | 🟡 中等 |
| **XxlJobConfig** | XXL-JOB 配置正确 | 🟡 中等 |
| **单元测试** | 核心 Service 有测试，测试可运行 | 🟡 中等 |

### 4.2 前端详细检查项

| 检查项 | 标准 | 严重程度 |
|-------|------|---------|
| **package.json** | 依赖完整，scripts 正确 | 🔴 严重 |
| **vite.config.js** | 代理配置正确，别名配置正确 | 🟡 中等 |
| **main.js** | ElementPlus 注册，VueRouter 配置正确 | 🔴 严重 |
| **路由配置** | 路由完整，路由守卫正确 | 🟡 中等 |
| **页面组件** | 各功能页面完整，代码合理 | 🟡 中等 |
| **API 封装** | axios 拦截器正确，请求/响应处理完善 | 🔴 严重 |
| **登录页面** | 登录/注册功能，Token 存储 | 🔴 严重 |
| **App.vue** | 布局合理，用户信息显示，登出功能 | 🟡 中等 |

### 4.3 通用检查项

| 检查项 | 标准 | 严重程度 |
|-------|------|---------|
| **Git 仓库** | .gitignore 正确，敏感信息不上传 | 🟡 中等 |
| **README.md** | 项目说明，快速启动，技术栈介绍 | 🟡 中等 |
| **代码注释** | 关键逻辑有注释，注释清晰 | 🟢 轻微 |
| **命名规范** | 变量/函数/类名符合规范 | 🟢 轻微 |

## 5. 检查执行流程

```
1. 确定检查类型和检查范围
   ↓
2. 按照检查清单逐项检查
   ↓
3. 记录发现的问题
   ↓
4. 评估问题严重程度
   ↓
5. 填写检查报告
   ↓
6. 报告给相关人员
   ↓
7. 跟踪问题修复进度
```
