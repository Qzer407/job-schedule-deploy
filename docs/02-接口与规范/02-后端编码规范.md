## 二、代码规范

### 2.1 命名规范

| 类型 | 命名规则 | 示例 |
|-----|---------|------|
| 类名 | 大驼峰 | `TaskController`, `TaskServiceImpl` |
| 方法名 | 小驼峰 | `createTask`, `getTaskById` |
| 变量名 | 小驼峰 | `taskName`, `taskList` |
| 常量 | 全大写下划线 | `MAX_RETRY_COUNT` |
| 包名 | 全小写 | `com.qzer.scheduler.controller` |

### 2.2 Controller 层规范

- 使用 `@RestController` 注解
- 使用 `@RequestMapping` 定义基础路径
- 遵循 RESTful API 设计原则
- 使用 `@Valid` 进行参数校验
- 返回 `ApiResponse<T>` 统一响应

```java
@RestController
@RequestMapping("/api/v1/task")
@RequiredArgsConstructor
public class TaskController {

    private final TaskService taskService;

    @PostMapping
    public ApiResponse<TaskInfo> createTask(@Valid @RequestBody TaskCreateRequest request) {
        TaskInfo task = taskService.createTask(request);
        return ApiResponse.success(task);
    }
}
```

### 2.3 Service 层规范

- 使用接口+实现类模式
- 使用 `@Service` 注解
- 使用 `@RequiredArgsConstructor` 注入依赖
- 使用 `@Slf4j` 记录日志
- 事务使用 `@Transactional` 注解

```java
public interface TaskService {
    TaskInfo createTask(TaskCreateRequest request);
    TaskInfo updateTask(Long id, TaskUpdateRequest request);

    TaskInfo getTaskById(Long id);

    void startTask(Long id);
    void stopTask(Long id);
    void triggerTask(Long id, String executorParam);
}

@Slf4j
@Service
@RequiredArgsConstructor
public class TaskServiceImpl implements TaskService {

    private final TaskInfoMapper taskInfoMapper;

    @Override
    @Transactional
    public TaskInfo createTask(TaskCreateRequest request) {
        // 实现逻辑
    }
}
```

### 2.5 Entity 层规范

- 使用 `@Data` 注解 (Lombok)
- 使用 `@TableName` 指定表名
- 使用 `@TableId` 指定主键
- 使用 `@TableField` 映射字段

```java
@Data
@TableName("t_task_info")
public class TaskInfo {

    @TableId(value = "id", type = IdType.AUTO)
    private Long id;

    @TableField("task_name")
    private String taskName;
}
```

### 2.6 DTO 层规范

- 请求 DTO 使用 `@Data` 注解
- 使用 Jakarta Validation 进行校验
- 响应 DTO 使用 `ApiResponse<T>` 包装

```java
@Data
public class TaskCreateRequest {

    @NotBlank(message = "任务名称不能为空")
    private String taskName;

    @NotBlank(message = "任务分组不能为空")
    private String taskGroup;
}

@Data
@NoArgsConstructor
@AllArgsConstructor
public class ApiResponse<T> {
    private Integer code;
    private String message;
    private T data;
    private Long timestamp;

    public static <T> ApiResponse<T> success(T data) {
        return new ApiResponse<>(200, "success", data, System.currentTimeMillis());
    }

    public static <T> ApiResponse<T> error(String message) {
        return new ApiResponse<>(500, message, null, System.currentTimeMillis());
    }
}
```

### 2.7 配置类规范

- 使用 `@Configuration` 注解
- 使用 `@Value` 读取配置
- XXL-JOB 配置使用 `@Bean` 返回 `XxlJobSpringExecutor`

```java
@Configuration
public class XxlJobConfig {

    @Value("${xxl.job.admin.addresses}")
    private String adminAddresses;

    @Bean
    public XxlJobSpringExecutor xxlJobExecutor() {
        XxlJobSpringExecutor executor = new XxlJobSpringExecutor();
        executor.setAdminAddresses(adminAddresses);
        // 其他配置...
        return executor;
    }
}
```

### 2.8 异常处理规范

- 使用 `@RestControllerAdvice` 全局处理异常
- 使用 `@ExceptionHandler` 捕获特定异常

```java
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(RuntimeException.class)
    public ApiResponse<Void> handleRuntimeException(RuntimeException ex) {
        log.error("运行时异常: {}", ex.getMessage(), ex);
        return ApiResponse.error(ex.getMessage());
    }
}
```

## 四、API 设计规范

|--------|------|------|
| `/api/v1/task` | POST | 创建任务 |
| `/api/v1/task` | GET | 查询任务列表 |
| `/api/v1/task/{id}` | GET | 查询任务详情 |

| `/api/v1/task/{id}/stop` | POST | 停止任务 |

## 五、日志规范

- 使用 `@Slf4j` 注解
- 日志级别：DEBUG < INFO < WARN < ERROR
- 关键操作记录 INFO 级别日志
- 异常记录 ERROR 级别日志

```java
@Slf4j
@Service
public class TaskServiceImpl {

    public void createTask(TaskCreateRequest request) {
        log.info("创建任务: {}", request.getTaskName());
    }
}
```
