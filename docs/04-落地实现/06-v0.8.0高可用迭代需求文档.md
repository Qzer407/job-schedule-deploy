## 三、数据库设计

### 3.1 集群节点表（t_cluster_node）

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| id | BIGINT | 主键 | AUTO_INCREMENT |
| node_id | VARCHAR(64) | 节点唯一标识 | NOT NULL, UNIQUE |
| node_name | VARCHAR(100) | 节点名称 | NOT NULL |
| node_ip | VARCHAR(45) | 节点IP地址 | NOT NULL |
| node_port | INT | 节点端口 | NOT NULL |
| status | TINYINT | 状态 | 0-离线,1-在线,2-故障 |
| role | VARCHAR(20) | 角色 | master,worker |
| last_heartbeat | DATETIME | 最后心跳时间 | NOT NULL |
| cpu_load | DECIMAL(5,2) | CPU负载 | |
| memory_usage | DECIMAL(5,2) | 内存使用率 | |
| create_time | DATETIME | 创建时间 | DEFAULT CURRENT_TIMESTAMP |
| update_time | DATETIME | 更新时间 | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP |

**索引：**
- PRIMARY KEY (id)
- UNIQUE KEY uk_node_id (node_id)
- INDEX idx_status (status)
- INDEX idx_last_heartbeat (last_heartbeat)

### 3.2 任务分片表（t_task_shard）

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| id | BIGINT | 主键 | AUTO_INCREMENT |
| task_id | BIGINT | 任务ID | NOT NULL |
| shard_key | VARCHAR(100) | 分片键 | |
| assigned_node_id | VARCHAR(64) | 分配节点ID | |
| shard_status | TINYINT | 分片状态 | 0-待分配,1-已分配,2-迁移中 |
| create_time | DATETIME | 创建时间 | DEFAULT CURRENT_TIMESTAMP |
| update_time | DATETIME | 更新时间 | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP |

**索引：**
- PRIMARY KEY (id)
- INDEX idx_task_id (task_id)
- INDEX idx_assigned_node (assigned_node_id)

### 3.3 集群事件表（t_cluster_event）

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| id | BIGINT | 主键 | AUTO_INCREMENT |
| event_type | VARCHAR(50) | 事件类型 | node_join,node_leave,failover,task_migrate |
| event_data | TEXT | 事件数据(JSON) | |
| event_time | DATETIME | 事件时间 | NOT NULL |
| source_node | VARCHAR(64) | 源节点 | |
| target_node | VARCHAR(64) | 目标节点 | |
| create_time | DATETIME | 创建时间 | DEFAULT CURRENT_TIMESTAMP |

**索引：**
- PRIMARY KEY (id)
- INDEX idx_event_type (event_type)
- INDEX idx_event_time (event_time)

## 四、接口设计

### 4.1 集群管理接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/cluster/nodes | 获取节点列表 |
| GET | /api/v1/cluster/nodes/{nodeId} | 获取节点详情 |
| DELETE | /api/v1/cluster/nodes/{nodeId} | 移除节点 |
| GET | /api/v1/cluster/status | 获取集群状态 |
| GET | /api/v1/cluster/events | 获取集群事件 |

### 4.2 任务分片接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/cluster/shards | 获取任务分片 |
| POST | /api/v1/cluster/shards/rebalance | 手动触发再平衡 |
| POST | /api/v1/cluster/shards/{taskId}/migrate | 迁移任务到指定节点 |

### 4.3 故障恢复接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/cluster/failures | 获取故障记录 |
| POST | /api/v1/cluster/failures/{id}/recover | 手动触发恢复 |

```
                    ┌─────────────────┐
                    │   负载均衡器     │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│   节点1       │   │   节点2       │   │   节点3       │
│   (Master)    │   │   (Worker)    │   │   (Worker)    │
│               │   │               │   │               │
│ - 任务分配    │   │ - 执行任务    │   │ - 执行任务    │
│ - 分片管理    │   │ - 心跳上报    │   │ - 心跳上报    │
│ - 健康检查    │   │ - 状态同步    │   │ - 状态同步    │
└───────────────┘   └───────────────┘   └───────────────┘
        │                    │                    │
        └────────────────────┼────────────────────┘
                             │
                    ┌────────▼────────┐
                    │   Redis         │
                    │ (共享数据)      │
                    └─────────────────┘
                             │
                    ┌────────▼────────┐
                    │   MySQL         │

                    └─────────────────┘
```

## 八、风险评估

| 风险点 | 影响 | 概率 | 应对措施 |
|--------|------|------|----------|
| 网络分区处理 | 高 | 中 | 使用 Raft/PAXOS 简化实现一致性 |
| 数据一致性 | 高 | 中 | 使用分布式锁和最终一致性 |
| 单点故障 | 高 | 低 | 多 Master 设计或快速故障转移 |
| 性能影响 | 中 | 中 | 异步通信和批量处理 |
