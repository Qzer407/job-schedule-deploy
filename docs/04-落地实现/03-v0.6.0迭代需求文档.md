## 3. 数据库设计

### 3.1 执行器分组表 (t_executor_group)
```sql
CREATE TABLE t_executor_group (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    group_name VARCHAR(100) NOT NULL COMMENT '分组名称',
    group_code VARCHAR(50) NOT NULL COMMENT '分组编码',
    description VARCHAR(500) COMMENT '分组描述',
    status TINYINT DEFAULT 1 COMMENT '状态 0-禁用 1-启用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_group_code (group_code)
) COMMENT='执行器分组表';
```

### 3.2 任务调度日志表 (t_schedule_log)
```sql
CREATE TABLE t_schedule_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    task_id BIGINT NOT NULL COMMENT '任务ID',
    task_name VARCHAR(200) COMMENT '任务名称',
    schedule_time DATETIME NOT NULL COMMENT '调度时间',
    trigger_type VARCHAR(50) COMMENT '触发类型 CRON/MANUAL/API',
    status TINYINT COMMENT '调度状态 0-失败 1-成功',
    error_message TEXT COMMENT '错误信息',
    cost_time BIGINT COMMENT '耗时(ms)',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_task_id (task_id),
    INDEX idx_schedule_time (schedule_time)
) COMMENT='任务调度日志表';
```

### 3.3 操作审计日志表 (t_audit_log)
```sql
CREATE TABLE t_audit_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT COMMENT '用户ID',
    username VARCHAR(100) COMMENT '用户名',
    operation VARCHAR(100) NOT NULL COMMENT '操作类型',
    module VARCHAR(100) COMMENT '操作模块',
    description VARCHAR(500) COMMENT '操作描述',
    request_method VARCHAR(20) COMMENT '请求方法',
    request_url VARCHAR(500) COMMENT '请求URL',
    request_params TEXT COMMENT '请求参数',
    response_result TEXT COMMENT '响应结果',
    ip_address VARCHAR(50) COMMENT 'IP地址',
    status TINYINT DEFAULT 1 COMMENT '状态 0-失败 1-成功',
    cost_time BIGINT COMMENT '耗时(ms)',
    error_message TEXT COMMENT '错误信息',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_create_time (create_time)
) COMMENT='操作审计日志表';
```

## 4. 接口设计

### 4.1 执行器分组接口
- `POST /api/v1/executor-group` - 创建分组
- `PUT /api/v1/executor-group/{id}` - 更新分组
- `DELETE /api/v1/executor-group/{id}` - 删除分组
- `GET /api/v1/executor-group/{id}` - 获取分组详情
- `GET /api/v1/executor-group` - 查询分组列表

### 4.2 调度日志接口
- `GET /api/v1/schedule-log` - 查询调度日志
- `GET /api/v1/schedule-log/{id}` - 获取调度日志详情

### 4.3 健康检查接口
- `GET /api/v1/health` - 系统健康检查

### 4.4 执行报告接口
- `GET /api/v1/report/execution` - 获取执行统计报告
- `GET /api/v1/report/execution/export` - 导出执行报告

### 4.5 审计日志接口
- `GET /api/v1/audit-log` - 查询审计日志
- `GET /api/v1/audit-log/{id}` - 获取审计日志详情
