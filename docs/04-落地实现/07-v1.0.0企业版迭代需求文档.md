## 3. 数据库设计

### 3.1 角色表 (t_role)
```sql
CREATE TABLE t_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_name VARCHAR(50) NOT NULL COMMENT '角色名称',
    role_code VARCHAR(50) NOT NULL COMMENT '角色编码',
    description VARCHAR(200) COMMENT '角色描述',
    status TINYINT DEFAULT 1 COMMENT '状态 0-禁用 1-启用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_role_code (role_code),
    INDEX idx_status (status)
) COMMENT='角色表';
```

### 3.2 权限表 (t_permission)
```sql
CREATE TABLE t_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    permission_name VARCHAR(100) NOT NULL COMMENT '权限名称',
    permission_code VARCHAR(100) NOT NULL COMMENT '权限编码',
    permission_type VARCHAR(20) NOT NULL COMMENT '权限类型 MENU-菜单 OPERATION-操作',
    parent_id BIGINT DEFAULT 0 COMMENT '父权限ID',
    path VARCHAR(200) COMMENT '路由路径',
    icon VARCHAR(50) COMMENT '图标',
    sort_order INT DEFAULT 0 COMMENT '排序',
    status TINYINT DEFAULT 1 COMMENT '状态 0-禁用 1-启用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_permission_code (permission_code),
    INDEX idx_parent_id (parent_id),
    INDEX idx_status (status)
) COMMENT='权限表';
```

### 3.3 用户角色关联表 (t_user_role)
```sql
CREATE TABLE t_user_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    role_id BIGINT NOT NULL COMMENT '角色ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_role (user_id, role_id),
    INDEX idx_user_id (user_id),
    INDEX idx_role_id (role_id)
) COMMENT='用户角色关联表';
```

### 3.4 角色权限关联表 (t_role_permission)
```sql
CREATE TABLE t_role_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_id BIGINT NOT NULL COMMENT '角色ID',
    permission_id BIGINT NOT NULL COMMENT '权限ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_role_permission (role_id, permission_id),
    INDEX idx_role_id (role_id),
    INDEX idx_permission_id (permission_id)
) COMMENT='角色权限关联表';
```

### 3.5 租户表 (t_tenant)
```sql
CREATE TABLE t_tenant (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_code VARCHAR(50) NOT NULL COMMENT '租户编码',
    tenant_name VARCHAR(100) NOT NULL COMMENT '租户名称',
    contact_name VARCHAR(50) COMMENT '联系人',
    contact_phone VARCHAR(20) COMMENT '联系电话',
    contact_email VARCHAR(100) COMMENT '联系邮箱',
    status TINYINT DEFAULT 1 COMMENT '状态 0-禁用 1-启用',
    expire_time DATETIME COMMENT '过期时间',
    max_users INT DEFAULT 10 COMMENT '最大用户数',
    max_tasks INT DEFAULT 100 COMMENT '最大任务数',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_tenant_code (tenant_code),
    INDEX idx_status (status)
) COMMENT='租户表';
```

### 3.6 OAuth2客户端表 (t_oauth_client)
```sql
CREATE TABLE t_oauth_client (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    client_id VARCHAR(100) NOT NULL COMMENT '客户端ID',
    client_name VARCHAR(100) NOT NULL COMMENT '客户端名称',
    provider VARCHAR(50) NOT NULL COMMENT '提供商 GITHUB/GITLAB/GOOGLE',
    client_secret VARCHAR(200) NOT NULL COMMENT '客户端密钥',
    redirect_uri VARCHAR(500) COMMENT '回调地址',
    scope VARCHAR(200) COMMENT '权限范围',
    status TINYINT DEFAULT 1 COMMENT '状态 0-禁用 1-启用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_client_id (client_id),
    INDEX idx_provider (provider)
) COMMENT='OAuth2客户端表';
```

### 3.7 OAuth2用户绑定表 (t_oauth_user)
```sql
CREATE TABLE t_oauth_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    provider VARCHAR(50) NOT NULL COMMENT '提供商',
    openid VARCHAR(100) NOT NULL COMMENT '第三方用户ID',
    nickname VARCHAR(100) COMMENT '昵称',
    avatar VARCHAR(500) COMMENT '头像',
    email VARCHAR(100) COMMENT '邮箱',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_provider_openid (provider, openid),
    INDEX idx_user_id (user_id)
) COMMENT='OAuth2用户绑定表';
```

### 3.8 API密钥表 (t_api_key)
```sql
CREATE TABLE t_api_key (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    api_key VARCHAR(100) NOT NULL COMMENT 'API密钥',
    api_secret VARCHAR(200) NOT NULL COMMENT 'API密钥',
    key_name VARCHAR(100) NOT NULL COMMENT '密钥名称',
    permissions TEXT COMMENT '权限列表',
    rate_limit INT DEFAULT 1000 COMMENT '限流次数/分钟',
    expire_time DATETIME COMMENT '过期时间',
    status TINYINT DEFAULT 1 COMMENT '状态 0-禁用 1-启用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_api_key (api_key),
    INDEX idx_user_id (user_id),
    INDEX idx_status (status)
) COMMENT='API密钥表';
```

### 3.9 API调用日志表 (t_api_log)
```sql
CREATE TABLE t_api_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    api_key_id BIGINT COMMENT 'API密钥ID',
    user_id BIGINT COMMENT '用户ID',
    endpoint VARCHAR(200) NOT NULL COMMENT '接口端点',
    method VARCHAR(10) NOT NULL COMMENT '请求方法',
    request_params TEXT COMMENT '请求参数',
    response_status INT COMMENT '响应状态码',
    cost_time BIGINT COMMENT '耗时(ms)',
    ip VARCHAR(50) COMMENT 'IP地址',
    user_agent VARCHAR(500) COMMENT 'User-Agent',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_api_key_id (api_key_id),
    INDEX idx_user_id (user_id),
    INDEX idx_create_time (create_time),
    INDEX idx_endpoint (endpoint)
) COMMENT='API调用日志表';
```

## 4. 接口设计

### 4.1 角色管理接口
- `POST /api/v1/role` - 创建角色
- `PUT /api/v1/role/{id}` - 更新角色
- `DELETE /api/v1/role/{id}` - 删除角色
- `GET /api/v1/role` - 查询角色列表
- `GET /api/v1/role/{id}` - 获取角色详情

### 4.2 权限管理接口
- `POST /api/v1/permission` - 创建权限
- `PUT /api/v1/permission/{id}` - 更新权限
- `DELETE /api/v1/permission/{id}` - 删除权限
- `GET /api/v1/permission` - 查询权限列表（树形）
- `GET /api/v1/permission/{id}` - 获取权限详情

### 4.3 用户角色接口
- `POST /api/v1/user/{userId}/role` - 分配角色
- `DELETE /api/v1/user/{userId}/role/{roleId}` - 移除角色
- `GET /api/v1/user/{userId}/role` - 查询用户角色

### 4.4 角色权限接口
- `POST /api/v1/role/{roleId}/permission` - 分配权限
- `DELETE /api/v1/role/{roleId}/permission/{permissionId}` - 移除权限
- `GET /api/v1/role/{roleId}/permission` - 查询角色权限

### 4.5 租户管理接口
- `POST /api/v1/tenant` - 创建租户
- `PUT /api/v1/tenant/{id}` - 更新租户
- `DELETE /api/v1/tenant/{id}` - 删除租户
- `GET /api/v1/tenant` - 查询租户列表
- `GET /api/v1/tenant/{id}` - 获取租户详情
- `POST /api/v1/tenant/{id}/switch` - 切换租户

### 4.6 OAuth2接口
- `GET /api/v1/oauth/providers` - 获取OAuth提供商列表
- `GET /api/v1/oauth/{provider}/authorize` - OAuth授权
- `GET /api/v1/oauth/{provider}/callback` - OAuth回调
- `POST /api/v1/oauth/{provider}/bind` - 绑定第三方账号
- `DELETE /api/v1/oauth/{provider}/unbind` - 解绑第三方账号

### 4.7 API密钥接口
- `POST /api/v1/api-key` - 创建API密钥
- `PUT /api/v1/api-key/{id}` - 更新API密钥
- `DELETE /api/v1/api-key/{id}` - 删除API密钥
- `GET /api/v1/api-key` - 查询API密钥列表
- `GET /api/v1/api-key/{id}` - 获取API密钥详情
- `POST /api/v1/api-key/{id}/refresh` - 刷新密钥

### 4.8 API日志接口
- `GET /api/v1/api-log` - 查询API调用日志
- `GET /api/v1/api-log/statistics` - 获取API调用统计

### 4.9 数据导出接口
- `GET /api/v1/export/task` - 导出任务列表
- `GET /api/v1/export/task-log` - 导出执行日志
- `GET /api/v1/export/alarm` - 导出告警记录
- `GET /api/v1/export/consume` - 导出消费日志
- `GET /api/v1/export/report` - 导出报表数据

### 4.10 国际化接口
- `GET /api/v1/i18n/{lang}` - 获取国际化资源

## 5. 权限模型设计

### 5.1 RBAC模型
```
User ──n── Role ──n── Permission
  │
  └── Tenant
```

### 5.2 权限类型
- **菜单权限**: 控制页面访问
- **操作权限**: 控制按钮操作（新增、编辑、删除、导出等）

### 5.3 预设角色
- **超级管理员**: 拥有所有权限
- **租户管理员**: 管理本租户的所有功能
- **运维人员**: 监控、告警、运维功能
- **开发人员**: 任务管理、工作流
- **普通用户**: 只读权限
