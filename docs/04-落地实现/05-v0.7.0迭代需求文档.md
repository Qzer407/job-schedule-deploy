## 3. 数据库设计

### 3.1 消息任务表 (t_message_task)
```sql
CREATE TABLE t_message_task (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    task_id BIGINT COMMENT '关联任务ID',
    task_name VARCHAR(200) COMMENT '任务名称',
    message_id VARCHAR(100) NOT NULL COMMENT '消息ID',
    queue_name VARCHAR(100) NOT NULL COMMENT '队列名称',
    message_type VARCHAR(50) NOT NULL COMMENT '消息类型 ASYNC/DELAY',
    payload TEXT COMMENT '消息载荷',
    status TINYINT DEFAULT 0 COMMENT '状态 0-待发送 1-已发送 2-消费中 3-成功 4-失败',
    delay_time BIGINT COMMENT '延迟时间(ms)',
    execute_time DATETIME COMMENT '执行时间',
    retry_count INT DEFAULT 0 COMMENT '重试次数',
    max_retry INT DEFAULT 3 COMMENT '最大重试次数',
    error_message TEXT COMMENT '错误信息',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_task_id (task_id),
    INDEX idx_message_id (message_id),
    INDEX idx_status (status),
    INDEX idx_create_time (create_time)
) COMMENT='消息任务表';
```

### 3.2 消费记录表 (t_consume_log)
```sql
CREATE TABLE t_consume_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    message_id VARCHAR(100) NOT NULL COMMENT '消息ID',
    queue_name VARCHAR(100) NOT NULL COMMENT '队列名称',
    consumer VARCHAR(100) COMMENT '消费者',
    consume_time DATETIME NOT NULL COMMENT '消费时间',
    status TINYINT COMMENT '状态 0-失败 1-成功',
    cost_time BIGINT COMMENT '耗时(ms)',
    error_message TEXT COMMENT '错误信息',
    retry_count INT COMMENT '当前重试次数',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_message_id (message_id),
    INDEX idx_consume_time (consume_time)
) COMMENT='消费记录表';
```

### 3.3 死信消息表 (t_dead_letter)
```sql
CREATE TABLE t_dead_letter (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    original_message_id VARCHAR(100) NOT NULL COMMENT '原消息ID',
    queue_name VARCHAR(100) NOT NULL COMMENT '原队列',
    dead_letter_queue VARCHAR(100) NOT NULL COMMENT '死信队列',
    payload TEXT COMMENT '消息载荷',
    reason VARCHAR(200) COMMENT '死信原因',
    error_message TEXT COMMENT '错误信息',
    is_reprocessed TINYINT DEFAULT 0 COMMENT '是否已重处理 0-否 1-是',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_original_message_id (original_message_id),
    INDEX idx_create_time (create_time)
) COMMENT='死信消息表';
```

## 4. 接口设计

### 4.1 消息任务接口
- `POST /api/v1/message/task` - 提交异步任务
- `POST /api/v1/message/task/delay` - 提交延迟任务
- `GET /api/v1/message/task` - 查询任务列表
- `GET /api/v1/message/task/{id}` - 获取任务详情
- `DELETE /api/v1/message/task/{id}` - 取消任务
- `POST /api/v1/message/task/{id}/retry` - 重试任务

### 4.2 队列管理接口
- `GET /api/v1/message/queue` - 查询队列列表
- `GET /api/v1/message/queue/{queueName}` - 获取队列详情
- `DELETE /api/v1/message/queue/{queueName}/purge` - 清空队列
- `GET /api/v1/message/queue/{queueName}/messages` - 查询队列消息

### 4.3 消费监控接口
- `GET /api/v1/message/consume/log` - 查询消费日志
- `GET /api/v1/message/consume/statistics` - 获取消费统计

### 4.4 死信队列接口
- `GET /api/v1/message/dead-letter` - 查询死信消息
- `POST /api/v1/message/dead-letter/{id}/reprocess` - 重处理死信
- `DELETE /api/v1/message/dead-letter/{id}` - 删除死信
- `DELETE /api/v1/message/dead-letter` - 批量删除

## 5. RabbitMQ 架构设计

### 5.1 交换机与队列

```
┌─────────────────────────────────────────────────────────────┐
│                         Exchange                             │
│  ┌──────────────────┐  ┌──────────────────┐                 │
│  │ task.exchange    │  │ delay.exchange   │                 │
│  │ (direct)         │  │ (direct)         │                 │
│  └────────┬─────────┘  └────────┬─────────┘                 │
└───────────┼──────────────────────┼───────────────────────────┘
            │                      │
            │                      │
┌───────────▼───────────┐  ┌──────▼──────────────┐
│ task.queue            │  │ delay.queue         │
│ (异步任务队列)        │  │ (延迟队列)          │
└───────────┬───────────┘  └──────┬──────────────┘
            │                      │ (TTL + dead-letter)
            │                      │
┌───────────▼───────────┐  ┌──────▼──────────────┐
│ task.queue.dlq        │  │ task.queue          │
│ (死信队列)            │  │ (延迟到期后转发)    │
└───────────────────────┘  └─────────────────────┘
```

### 5.2 队列配置

| 队列名称 | 类型 | 说明 | 死信队列 |
|----------|------|------|----------|
| task.queue | 主队列 | 异步任务执行 | task.queue.dlq |
| task.queue.dlq | 死信队列 | 处理失败的消息 | - |
| delay.queue | 延迟队列 | 临时存储延迟消息 | task.queue |
