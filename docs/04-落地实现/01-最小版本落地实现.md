### 3.2 实体类详细定义

#### TaskInfo.java
- id: Long - 主键
- taskName: String - 任务名称（必填）
- taskGroup: String - 任务分组（必填）
- cronExpression: String - Cron表达式
- executorHandler: String - 执行器处理器（必填）
- executorParam: String - 执行参数
- shardingTotal: Integer - 分片总数（默认1）
- shardingParam: String - 分片参数
- retryCount: Integer - 重试次数（默认3）
- alarmStatus: Integer - 告警状态（0关闭/1开启）
- status: Integer - 任务状态（0停用/1启用）
- description: String - 任务描述
- createTime: LocalDateTime - 创建时间
- updateTime: LocalDateTime - 更新时间

#### TaskLog.java
- id: Long - 主键
- taskId: Long - 任务ID
- taskName: String - 任务名称
- executorAddress: String - 执行器地址
- shardingIndex: Integer - 分片索引
- shardingParam: String - 分片参数
- triggerTime: LocalDateTime - 触发时间
- startTime: LocalDateTime - 开始时间
- endTime: LocalDateTime - 结束时间
- duration: Long - 执行耗时(毫秒)
- status: Integer - 执行状态（0失败/1成功）
- errorMsg: String - 错误信息
- createTime: LocalDateTime - 创建时间

### 3.3 Service层实现要点

#### TaskServiceImpl核心方法

1. **createTask**: 创建任务，插入数据库
2. **updateTask**: 更新任务，检查任务存在性
3. **deleteTask**: 删除任务
4. **getTaskById**: 根据ID查询任务
5. **listTasks**: 分页查询任务列表，支持条件筛选
6. **startTask**: 启动任务（更新状态为1）
7. **stopTask**: 停止任务（更新状态为0）
8. **triggerTask**: 手动触发任务，调用XXL-JOB API

#### MonitorServiceImpl核心方法

1. **getStatistics**: 获取统计概览
   - totalTaskCount: 总任务数
   - runningTaskCount: 运行中任务数
   - successRate: 成功率
   - avgDuration: 平均执行耗时
   - todayExecuteCount: 今日执行次数
   - todayFailCount: 今日失败次数

2. **getTaskLogs**: 分页查询任务执行日志
3. **getAlarmRecords**: 分页查询告警记录
4. **handleAlarm**: 处理告警
5. **triggerAlarm**: 触发告警

## 四、数据库实现

### 4.1 建表SQL

#### 步骤2：执行SQL脚本

```bash
# 导入表结构
mysql -u root -p < deploy/config/schema.sql
```

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS task_scheduler
    DEFAULT CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

USE task_scheduler;

-- 任务配置表
CREATE TABLE `t_task_info` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `task_name` VARCHAR(200) NOT NULL COMMENT '任务名称',
  `task_group` VARCHAR(100) NOT NULL COMMENT '任务分组',
  `cron_expression` VARCHAR(100) DEFAULT NULL COMMENT 'Cron表达式',
  `executor_handler` VARCHAR(200) NOT NULL COMMENT '执行器处理器',
  `executor_param` VARCHAR(500) DEFAULT NULL COMMENT '执行参数',
  `sharding_total` INT DEFAULT 1 COMMENT '分片总数',
  `sharding_param` VARCHAR(500) DEFAULT NULL COMMENT '分片参数',
  `retry_count` INT DEFAULT 3 COMMENT '重试次数',
  `alarm_status` TINYINT DEFAULT 1 COMMENT '告警状态(0关闭/1开启)',
  `status` TINYINT DEFAULT 0 COMMENT '任务状态(0停用/1启用)',
  `description` VARCHAR(500) DEFAULT NULL COMMENT '任务描述',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_task_group` (`task_group`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='任务配置表';

-- 任务执行日志表
CREATE TABLE `t_task_log` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `task_id` BIGINT NOT NULL COMMENT '任务ID',
  `task_name` VARCHAR(200) DEFAULT NULL COMMENT '任务名称',
  `executor_address` VARCHAR(200) DEFAULT NULL COMMENT '执行器地址',
  `sharding_index` INT DEFAULT 0 COMMENT '分片索引',
  `sharding_param` VARCHAR(500) DEFAULT NULL COMMENT '分片参数',
  `trigger_time` DATETIME DEFAULT NULL COMMENT '触发时间',
  `start_time` DATETIME DEFAULT NULL COMMENT '开始时间',
  `end_time` DATETIME DEFAULT NULL COMMENT '结束时间',
  `duration` BIGINT DEFAULT 0 COMMENT '执行耗时(毫秒)',
  `status` TINYINT DEFAULT NULL COMMENT '执行状态(0失败/1成功)',
  `error_msg` TEXT COMMENT '错误信息',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_task_id` (`task_id`),
  KEY `idx_trigger_time` (`trigger_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='任务执行日志表';

-- 告警记录表
CREATE TABLE `t_alarm_record` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `task_id` BIGINT DEFAULT NULL COMMENT '任务ID',
  `task_name` VARCHAR(200) DEFAULT NULL COMMENT '任务名称',
  `alarm_type` VARCHAR(50) DEFAULT NULL COMMENT '告警类型',
  `alarm_content` TEXT COMMENT '告警内容',
  `alarm_channel` VARCHAR(50) DEFAULT NULL COMMENT '告警渠道',
  `alarm_time` DATETIME DEFAULT NULL COMMENT '告警时间',
  `status` TINYINT DEFAULT 0 COMMENT '处理状态(0未处理/1已处理)',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_task_id` (`task_id`),
  KEY `idx_alarm_time` (`alarm_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='告警记录表';
```

## 七、常见问题

1. **XXL-JOB调度中心未启动**：需要先启动XXL-JOB Admin
2. **数据库连接失败**：检查MySQL服务是否启动，配置是否正确
3. **端口被占用**：检查8080端口是否被其他应用占用
