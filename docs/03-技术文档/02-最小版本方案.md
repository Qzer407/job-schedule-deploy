# 最小版本方案（MVP）

## 一、MVP目标

最小版本方案旨在快速搭建一个可运行的分布式任务调度平台，聚焦核心功能，确保项目能够快速启动并验证技术可行性。

## 二、最小功能范围

### 2.1 核心功能

| 功能模块 | 功能点 | 优先级 | 说明 |
|---------|--------|--------|------|
| **任务管理** | 任务创建 | P0 | 支持Cron表达式、任务分组、执行器配置 |
| | 任务列表查询 | P0 | 分页查询、支持条件筛选 |
| | 任务详情查询 | P0 | 查看任务配置详情 |
| | 任务更新 | P0 | 修改任务配置 |
| | 任务删除 | P0 | 删除任务 |
| | 任务启动/停止 | P0 | 控制任务状态 |
| | 手动触发任务 | P0 | 立即执行任务 |
| **任务执行** | XXL-JOB集成 | P0 | 调度中心、执行器配置 |
| | 示例任务处理器 | P0 | 演示任务执行 |
| **基础监控** | 执行日志查询 | P1 | 查看任务执行记录 |
| | 统计概览 | P1 | 总任务数、运行数、成功率 |

### 2.2 非功能需求

| 需求项 | 最小要求 |
|--------|---------|
| 调度响应时间 | ≤ 500ms |
| 告警功能 | 暂不实现（后续版本） |
| 用户权限 | 暂不实现（后续版本） |
| 前端界面 | 暂不实现（后续版本） |

## 五、数据库设计（MVP）

### 5.1 核心表

MVP阶段仅保留最核心的3张表：

| 表名 | 说明 | 字段数 |
|-----|------|--------|
| `t_task_info` | 任务配置表 | 13 |
| `t_task_log` | 任务执行日志表 | 12 |
| `t_alarm_record` | 告警记录表 | 9 |

### 5.2 表结构

#### 任务配置表 (t_task_info)

```sql
CREATE TABLE `t_task_info` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `task_name` VARCHAR(200) NOT NULL COMMENT '任务名称',
  `task_group` VARCHAR(100) NOT NULL COMMENT '任务分组',
  `cron_expression` VARCHAR(100) DEFAULT NULL COMMENT 'Cron表达式',
  `executor_handler` VARCHAR(200) NOT NULL COMMENT '执行器处理器',
  `executor_param` VARCHAR(500) DEFAULT NULL COMMENT '执行参数',
  `sharding_total` INT DEFAULT 1 COMMENT '分片总数',
  `sharding_param` VARCHAR(500) DEFAULT NULL COMMENT '分片参数',
  `retry_count` INT DEFAULT 3 COMMENT '重试次数',
  `alarm_status` TINYINT DEFAULT 1 COMMENT '告警状态(0关闭/1开启)',
  `status` TINYINT DEFAULT 0 COMMENT '任务状态(0停用/1启用)',
  `description` VARCHAR(500) DEFAULT NULL COMMENT '任务描述',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_task_group` (`task_group`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='任务配置表';
```

#### 任务执行日志表 (t_task_log)

```sql
CREATE TABLE `t_task_log` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `task_id` BIGINT NOT NULL COMMENT '任务ID',
  `task_name` VARCHAR(200) DEFAULT NULL COMMENT '任务名称',
  `executor_address` VARCHAR(200) DEFAULT NULL COMMENT '执行器地址',
  `sharding_index` INT DEFAULT 0 COMMENT '分片索引',
  `sharding_param` VARCHAR(500) DEFAULT NULL COMMENT '分片参数',
  `trigger_time` DATETIME DEFAULT NULL COMMENT '触发时间',
  `start_time` DATETIME DEFAULT NULL COMMENT '开始时间',
  `end_time` DATETIME DEFAULT NULL COMMENT '结束时间',
  `duration` BIGINT DEFAULT 0 COMMENT '执行耗时(毫秒)',
  `status` TINYINT DEFAULT NULL COMMENT '执行状态(0失败/1成功)',
  `error_msg` TEXT COMMENT '错误信息',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_task_id` (`task_id`),
  KEY `idx_trigger_time` (`trigger_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='任务执行日志表';
```

## 六、API接口设计（MVP）

### 6.1 任务管理接口

| API路径 | 方法 | 功能 |
|--------|------|------|
| `/api/v1/task` | POST | 创建任务 |

## 七、快速启动指南

### 7.1 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| JDK | 21+ | Java运行环境 |
| Maven | 3.8+ | 项目构建 |
| MySQL | 8.0+ | 数据库 |

### 7.2 启动步骤

#### 步骤1：创建数据库

```sql
CREATE DATABASE IF NOT EXISTS task_scheduler DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE task_scheduler;
```

#### 步骤2：执行SQL脚本

```bash
# 导入表结构
mysql -u root -p task_scheduler < sql/tables_mysql.sql
```

#### 步骤3：配置数据库连接

编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/task_scheduler?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai
    username: root
    password: your_password
    driver-class-name: com.mysql.cj.jdbc.Driver
```

#### 步骤4：配置XXL-JOB

```yaml
xxl:
  job:
    admin:
      addresses: http://127.0.0.1:8080/xxl-job-admin
    executor:
      appname: xxl-job-executor
      ip:
      port: 9999
      logpath: /tmp/xxl-job/jobhandler
      logretentiondays: 30
    accessToken: default_token
```

#### 步骤5：启动应用

```bash
cd backend
mvn spring-boot:run
```

### 7.3 验证

| 验证项 | 验证方式 | 预期结果 |
|--------|---------|---------|

1. **XXL-JOB部署**：MVP阶段需要单独部署XXL-JOB调度中心

4. **安全考虑**：MVP版本暂未实现权限控制，生产环境需自行添加
