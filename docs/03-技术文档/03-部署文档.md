# 分布式定时任务调度平台 - 部署文档

## 1. 环境要求

### 1.1 运行环境

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 21+ | 后端运行环境 |
| Node.js | 18+ | 前端构建工具 |
| MySQL | 8.0+ | 数据库 |
| XXL-JOB Admin | 2.4.x | 任务调度平台 |
| Docker | 20.x+ | 容器化部署（可选） |

### 1.2 系统资源

| 资源 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核+ |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 20GB | 50GB+ |

## 2. 快速部署

### 2.1 Docker Compose 部署（推荐）

```bash
# 进入部署目录
cd deploy

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps
```

### 2.2 手动部署

#### 2.2.1 后端部署

```bash
# 进入后端目录
cd backend

# 构建项目
mvn clean package -DskipTests

# 运行项目
java -jar target/job-scheduler-0.1.0.jar
```

#### 2.2.2 前端部署

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 开发模式
npm run dev

# 生产构建
npm run build
```

## 3. 环境配置

### 3.1 数据库配置

```bash
# 创建数据库
CREATE DATABASE task_scheduler DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 执行初始化脚本
source deploy/config/schema.sql
```

### 3.2 XXL-JOB 配置

1. 部署 XXL-JOB Admin（参考官方文档）
2. 添加执行器配置
3. 注册执行器

### 3.3 环境变量配置

创建 `.env` 文件（参考 `.env.example`）：

```bash
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=task_scheduler
DB_USERNAME=root
DB_PASSWORD=your_password

# XXL-JOB 配置
XXL_JOB_ADMIN_ADDRESSES=http://127.0.0.1:8081/xxl-job-admin
XXL_JOB_ACCESS_TOKEN=your_token

# JWT 配置
JWT_SECRET=your_jwt_secret_min_32_chars
JWT_EXPIRATION=86400000
```

## 4. Docker 部署详解

### 4.1 docker-compose.yml

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: task-scheduler-mysql
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: task_scheduler
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./config/schema.sql:/docker-entrypoint-initdb.d/schema.sql
    command: --default-authentication-plugin=mysql_native_password

  backend:
    build: ./backend
    container_name: task-scheduler-backend
    environment:
      DB_HOST: mysql
      DB_PORT: 3306
      DB_NAME: task_scheduler
      DB_USERNAME: root
      DB_PASSWORD: root_password
      XXL_JOB_ADMIN_ADDRESSES: http://xxl-job:8080/xxl-job-admin
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - xxl-job

  frontend:
    build: ./frontend
    container_name: task-scheduler-frontend
    ports:
      - "3000:80"

  xxl-job:
    image: xuxueli/xxl-job-admin:2.4.0
    container_name: xxl-job-admin
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/xxl_job?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: root_password
    ports:
      - "8081:8080"
    depends_on:
      - mysql

volumes:
  mysql_data:
```

### 4.2 后端 Dockerfile

```dockerfile
FROM eclipse-temurin:21-jre-alpine

WORKDIR /app

COPY target/job-scheduler-*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 4.3 前端 Dockerfile

```dockerfile
FROM node:18-alpine AS builder

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

## 6. 常见问题

### 6.1 数据库连接失败

- 检查数据库服务是否启动
- 验证用户名密码是否正确
- 确认数据库已创建

### 6.2 XXL-JOB 无法注册执行器

- 确认 XXL-JOB Admin 地址配置正确
- 检查网络连通性
- 验证 accessToken 是否匹配

### 6.3 前端无法访问后端

- 确认后端服务已启动
- 检查跨域配置
- 验证 API 地址是否正确
