## 四、数据库变更

### 4.1 新增表

```sql
-- 子任务表
CREATE TABLE IF NOT EXISTS `t_sub_task` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `parent_task_id` BIGINT NOT NULL,
    `sub_task_name` VARCHAR(200) NOT NULL,
    `executor_handler` VARCHAR(200) NOT NULL,
    `executor_param` VARCHAR(500),
    `retry_count` INT DEFAULT 3,
    `order_index` INT DEFAULT 0,
    `status` TINYINT DEFAULT 1,
    `description` VARCHAR(500),
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME ON UPDATE CURRENT_TIMESTAMP,
    INDEX `idx_parent_task_id` (`parent_task_id`),
    CONSTRAINT `fk_sub_task_parent_task_id` FOREIGN KEY (`parent_task_id`) REFERENCES `t_task_info`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 工作流模板表
CREATE TABLE IF NOT EXISTS `t_workflow_template` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `template_name` VARCHAR(200) NOT NULL,
    `template_desc` VARCHAR(500),
    `template_type` VARCHAR(50),
    `template_config` TEXT,
    `status` TINYINT DEFAULT 1,
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME ON UPDATE CURRENT_TIMESTAMP,
    INDEX `idx_template_name` (`template_name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 五、接口清单

| 接口名称 | HTTP方法 | 路径 | 说明 |
|---------|----------|------|------|
| 查询子任务列表 | GET | /api/v1/subtasks/parent/{parentTaskId} | 查询父任务下的子任务 |
| 查询子任务详情 | GET | /api/v1/subtasks/{id} | 查询单个子任务 |
| 创建子任务 | POST | /api/v1/subtasks/parent/{parentTaskId} | 创建子任务 |
| 批量创建子任务 | POST | /api/v1/subtasks/parent/{parentTaskId}/batch | 批量创建 |
| 更新子任务 | PUT | /api/v1/subtasks/{id} | 更新子任务 |
| 删除子任务 | DELETE | /api/v1/subtasks/{id} | 删除子任务 |
| 执行子任务 | POST | /api/v1/subtasks/parent/{parentTaskId}/execute | 执行子任务 |
| 查询模板列表 | GET | /api/v1/workflow-templates | 查询模板列表 |
| 查询模板详情 | GET | /api/v1/workflow-templates/{id} | 查询模板详情 |
| 创建模板 | POST | /api/v1/workflow-templates | 创建模板 |
| 更新模板 | PUT | /api/v1/workflow-templates/{id} | 更新模板 |
| 删除模板 | DELETE | /api/v1/workflow-templates/{id} | 删除模板 |
| 从模板创建工作流 | POST | /api/v1/workflow-templates/{id}/apply | 应用模板 |

## 六、技术亮点

1. **子任务管理**：支持任务嵌套，一个父任务可以包含多个子任务，按顺序执行
2. **工作流模板**：支持模板化管理，快速创建相似工作流
3. **DAG可视化**：工作流依赖可视化编辑器，直观展示任务关系
